# 📚 코딩 테스트 학습 복습 노트 (Java)

## 1. 오늘 학습한 핵심 개념

### 🛠 입출력 시스템 (I/O)

- **BufferedReader**: `Scanner`보다 훨씬 빠른 Java의 표준 입력 방식입니다.
- **StringTokenizer**: 한 줄에 공백을 두고 여러 숫자가 들어올 때 필수적인 도구입니다.
- **다중 라인 입력**: `br.readLine()`은 한 줄만 읽으므로, 입력 데이터가 줄바꿈되어 있다면 각 줄마다 `new StringTokenizer`를 하거나 `readLine()`을 새로 호출해야 합니다.

### 💡 알고리즘 및 로직

- **단위 환산 (Normalization)**: 시(Hour)와 분(Minute)을 하나의 단위인 '분'으로 합쳐서 계산하면 복잡한 `if`문을 획기적으로 줄일 수 있습니다.
- **나머지 연산자 (`%`)**: 24시간 주기를 계산할 때 `totalMin % 24`를 사용하면 별도의 조건문 없이도 시간을 0~23 사이로 고정할 수 있습니다.
- **삼항 연산자**: 간단한 조건 분기는 `(조건) ? 참 : 거짓`으로 가독성 있게 표현할 수 있습니다.

## 3. 발생했던 에러와 해결책 (Troubleshooting)

> ❗ **NoSuchElementException**
>
> - **현상**: `st.nextToken()` 호출 시 데이터가 없어서 에러 발생.
> - **원인**: 입력값이 두 줄인데 첫 번째 줄에서만 데이터를 찾으려고 함.
> - **해결**: 다음 줄을 `br.readLine()`으로 새로 읽어들여 해결.
