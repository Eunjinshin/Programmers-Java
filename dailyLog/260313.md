## 📚 1. 오늘 학습 내용 이론 정리

### ① 산술 연산을 이용한 데이터 분해 (Digit Extraction)

문자열 변환(`String.valueOf`) 없이 숫자 자체를 산술적으로 다루는 법을 학습했습니다.

- **Modulo (`%`):** 특정 숫자로 나눈 나머지를 구합니다. `N % 10`은 항상 1의 자리 숫자를 추출합니다.
    
- **Division (`/`):** 나눗셈의 몫을 구합니다. `N / 10`은 1의 자리를 제거하고 숫자를 한 칸씩 뒤로 밀어냅니다.
    
- **핵심:** 문자열 변환은 새로운 객체를 생성하므로 메모리 비용이 발생하지만, 산술 연산은 CPU 기본 연산이라 매우 빠르고 효율적입니다.
    

### ② 카운팅 배열과 상태 관리 (Counting & State)

데이터의 존재 여부나 빈도를 배열의 **인덱스**와 매핑하는 기법입니다.

- **Direct Mapping:** 숫자 `5`가 나왔다면 `arr[5]`의 값을 조작합니다.
    
- **Distinct Count (중복 제거):** `boolean[]` 배열을 사용하여 "한 번이라도 불이 켜졌는가?"를 체크합니다. 이는 자료구조 `Set`의 가장 원시적이고 빠른 형태입니다.
    

### ③ 데이터 정규화와 부동 소수점 오차 (Floating Point)

실무에서도 가장 사고가 많이 터지는 수치 연산 영역입니다.

- **정수 나눗셈의 함정:** 자바에서 `int / int`는 결과도 `int`입니다. 소수점 이하를 가차 없이 버립니다.
    
- **형 변환 (Casting):** 연산 중 하나라도 `double`이 섞이면 전체 결과가 `double`로 승격(Promotion)됩니다.
    
- **오차 허용 범위:** 컴퓨터는 2진수로 소수를 표현하기 때문에 미세한 오차가 발생할 수밖에 없으며(IEEE 754), 이를 위해 코딩 테스트에서는 보통 $10^{-7}$ 이상의 정밀도를 요구합니다.
