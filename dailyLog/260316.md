# 📑 코딩 테스트 학습 리포트 (2026.03.16)

## 1. 오늘 학습 핵심 이론: 매개 변수 탐색 (Parametric Search)

오늘 다룬 '나무 자르기'와 '랜선 자르기'는 **이분 탐색**을 활용한 최적화 기법의 정수입니다.

### 🔍 개념 정의
최적화 문제(최댓값이나 최솟값을 찾는 문제)를 **결정 문제(Yes/No)**로 바꾸어 푸는 기법입니다.
- **기존 문제:** "N개를 만들기 위한 **최대** 길이는 얼마인가?"
- **결정 문제:** "길이가 $X$일 때, N개 이상을 만들 수 있는가? (**Yes/No**)"



### ⚙️ 핵심 로직 (Logic Flow)
1. **범위 설정:** 정답 후보의 최소(`low`)와 최대(`high`)를 설정합니다.
2. **중간값 검증:** `mid = (low + high) / 2`로 잘랐을 때의 결과값을 계산합니다.
3. **상태 변화에 따른 이동:**
   - **충분할 때 (Sum >= Target):** 일단 정답 후보로 기록(`result = mid`)하고, 더 좋은 답을 찾기 위해 `low = mid + 1`.
   - **부족할 때 (Sum < Target):** 기준이 너무 높으니 낮추기 위해 `high = mid - 1`.

| 항목 | 평가 | 세부 의견 |
| :--- | :---: | :--- |

| **언어 활용능력** | **B** | `BufferedReader` 사용 등 효율적 I/O는 좋으나, 자료형 선택(`int` vs `long`)에서 실수가 잦음. |

## 🛠️ 3. 오늘 보충 및 향후 학습 로드맵

### 🚩 오늘 놓친 '디테일' (Must Review)
1. **자료형의 한계:** 20억 이상의 합계가 예상될 땐 주저 없이 `long`을 사용하세요.
2. **Divide by Zero:** 이분 탐색에서 `low`를 0으로 잡으면 `mid`가 0이 되어 프로그램이 터질 수 있습니다. 랜선 길이는 최소 1이므로 `low = 1`이 안전합니다.
3. **입력 파싱:** `st.nextToken()`(공백 기반)과 `br.readLine()`(줄바꿈 기반)의 차이를 명확히 인지해야 합니다.
