## 냉정한 부족한 점 ❌

### 1. 초기화 타이밍을 놓쳤다

```java
// 반복문 밖에 선언
Queue<int[]> queue = new ArrayDeque<>();
PriorityQueue<Integer> maxHeap = new PriorityQueue<>();
```

> 이건 단순 실수가 아니에요.  
> **"테스트케이스마다 초기화가 필요하다"** 는 걸 설계 단계에서 인지 못 한 거예요.  
> 앞으로 테스트케이스가 여러 개인 문제는  
> **"매번 초기화할 변수가 뭔지"** 를 먼저 체크하는 습관이 필요해요.

### 2. while 조건 방향을 또 틀렸다

```java
while(queue.isEmpty())   // 1회차 ❌
while(!queue.isEmpty())  // 2회차 ✅
```

> 1874에서도 while 조건을 틀렸었죠.  
> **2회 연속 같은 실수**예요.  
> 코드 작성 전에 손으로 트레이싱하는 습관, 아직 안 잡혀있어요.

### 3. 정렬 먼저 하려는 습관이 있어요

> 첫 설계에서 **"정렬을 먼저 해야 할 것 같아"** 라고 하셨는데,  
> 시뮬레이션 문제에서 정렬은 순서를 망가뜨려요.  
> 문제 유형을 파악하는 훈련이 필요해요:

```
정렬 문제    → 순서를 바꾸는 게 목적
시뮬레이션   → 주어진 순서대로 동작을 재현하는 게 목적
```

### 4. return vs break 차이를 몰랐다

```java
return;  // 메서드 전체 종료
break;   // 현재 반복문만 탈출
```

> 3년차라면 이 차이는 즉시 나와야 해요.  
> `continue`, `break`, `return` 세 가지 정확한 동작 범위를 다시 정리하세요.
