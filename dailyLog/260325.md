## 잘한 점 ✅

- **손으로 슈도코드를 먼저 작성**한 것 (이전 문제 대비 큰 발전!)
- **원형 큐 개념**을 Queue 시뮬레이션으로 바꿔서 이해한 것
- **출력 형식**을 정확히 파악하고 구현한 것 (`<숫자, 숫자>`)
- **StringTokenizer 사용법**을 이제 안정적으로 쓰는 것

---

## 냉정한 부족한 점 ❌

### 1. 여전히 입력 읽는 순서를 틀렸다

```java
StringTokenizer st = new StringTokenizer(br.readLine());
int N = Integer.parseInt(br.readLine());  // ❌ 또 다른 줄
int K = Integer.parseInt(st.nextToken());
```

> **3문제 연속** 같은 실수예요.  
> 1966에서도, 1158에서도 똑같은 패턴.  
> 이건 단순 실수가 아니라 **입력 형식 분석 습관의 문제**예요.

### 2. poll() 중복 호출

```java
for (int j = 0; j < K - 1; j++) {
    queue.poll();           // 1번째
    queue.offer(queue.poll()); // 2번째
}
```

> 코드 작성 전에 **"poll() 몇 번 하는지"** 를 세어보는 습관이 없어요.  
> 이것도 손 트레이싱 부족의 신호예요.

### 3. 출력 형식을 한 번에 못 잡았다

> 문제에서 `<3, 6, 2, 7, 5, 1, 4>` 형태를 명시했는데  
> 처음엔 그냥 숫자만 출력했어요.  
> **문제 요구사항을 꼼꼼히 읽는 습관** 필요해요.
